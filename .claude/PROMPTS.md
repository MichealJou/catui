## 🎯 高级提示技巧

### 1. 组合 Skills

```
同时使用多个 skill 解决问题：

示例：
"使用 vue skill 和 ui-design-system skill 帮我创建一个数据表格组件，
要求支持虚拟滚动、固定列、排序功能。"
```

### 2. 逐步引导

```
对于复杂任务，分步进行：

"我想实现 [功能]，请帮我：
1. 先分析现有代码结构
2. 设计实现方案
3. 逐步实现每个模块
4. 最后进行整合测试"
```

### 3. 上下文提供

```
提供足够的上下文信息：

"在 packages/ctable/src/core/ 目录下，我想重构 Renderer 类。
当前实现有性能问题，数据量超过 1000 行时渲染变慢。
请参考同目录下的 CanvasRenderer.ts 的优化思路。"
```

### 4. 明确约束

```
明确说明约束条件：

"创建一个表格组件，但有以下约束：
- 必须使用 Composition API
- 不能使用 any 类型
- 样式只能用 Less
- 需要支持 TypeScript 严格模式
- 要考虑大列表性能（10万+ 行）"
```

### 5. 迭代优化

```

迭代式改进：

"请帮我优化这段代码：
[代码]

第一次迭代：先优化可读性
第二次迭代：再优化性能
第三次迭代：最后完善类型定义"
```

## 🎓 提示词最佳实践

### ✅ 好的提示词

```
❌ 模糊：帮我优化代码

✅ 明确：使用 vue skill 优化 CanvasTable.vue 的渲染性能，
       目标是将 10 万行数据的渲染时间从 5 秒降低到 1 秒以内。
       当前使用的是 Canvas 2D 渲染，请参考 VirtualScroll.ts 的实现。
```

### ✅ 好的提示词

```
❌ 不足：创建一个按钮组件

✅ 完善：创建一个 Vue 3 按钮组件，要求：
       - 使用 <script setup lang="ts">
       - 支持 type: 'primary' | 'default' | 'danger'
       - 支持 size: 'small' | 'medium' | 'large'
       - 支持 loading 状态
       - 支持 disabled 状态
       - 使用 Less 样式，参考 Element Plus 的设计
       - 添加完整的 TypeScript 类型
       - 添加 JSDoc 注释
```
